ReactiveX 学习笔记

这篇文章是 ReactiveX 系列的学习笔记，ReactiveX 包含多语言的 Rx (全称 Reactive Extension) 版本，比如 rxjs, rxjava, rxgo 等，基本上属于可以融会贯通，举一反三的技术栈，对多语言场景十分友好。此文基于 RxJs 5, 不同语言和版本的行为可能有差异。

Reactive 相关的技术栈在前端领域已经得到了广泛的应用，属于前端人员技能必修课，在安卓领域也已经被广泛使用了很久。然而 Reactive 在传统的服务端领域却一直不温不火，国内除了阿里这种有足够人力和技术积累的互联网公司在使用，其它使用者寥寥无几，搜索 RxJava 也能在搜索栏发现获赞颇高的唱衰信息，比如 我为什么不再推荐RxJava。就我个人看来，造成这种现象的原因主要有二：

  • Reactive 技术天然适用于事件属性较强的场景，前端和移动端比较契合
  • 新技术在成熟领域发展的阵痛期

一件非常有意思的事情是，现在的前后端语言特性正在逐渐靠拢，java 语言特性包含越来越多 FP 和 Stream 甚至 Module，ts 也将 OOP 和 IOC 等应用得淋漓尽致。私以为，Angular 部分不太受前端人员待见的特性将是前端的发展趋势，从 vue 逐渐支持 ts 便可见端倪。而至于说到 Reactive 在服务端所受的诟病，主要聚焦在高昂的学习成本和困难的问题定位方式。是不是感觉很熟悉？这正是每个人在刚接触到多线程、高并发的时候遇到的问题，而响应式编程思想的转换成本与从面向过程转向 OOD 时的不适应也如出一辙。

不同的编程范式之争由来已久，OOD，FP，RP 各具优势，学习曲线的陡峭也意味着融会贯通之后的随心所欲。而作为一名软件开发人员，即时工作中很少用到某种编程思想或范式，广积粮、高筑墙也是极好的。但也要避免无意义的炫技，新手总喜欢做加法，强调各种高级特性和设计，而大师总数能够大道至简，无招胜有招。千万不能一开始就追求无招，而忽略了无招的前提和基础是熟练掌握各种招式。

本文的目的主要为了梳理知识，会记录一些需要学习的技能和链接，技术细节不在讨论范围之内。
